# 🎯 مهمة إعادة هيكلة لوحة تحكم خدوم (khadum.app)

## السياق الحرج — اقرأه بعناية قبل أي شيء

أنت تعمل على منصة **خدوم (khadum.app)** — سوق خدمات مصغرة سعودي يعمل عبر واتساب.

**المنصة تعمل في الإنتاج حالياً** وتعالج أموالاً حقيقية عبر Tap Payments (Live Mode).

### التقنيات
- Next.js 15 (App Router)
- PostgreSQL + Drizzle ORM
- Tailwind CSS v4
- Tap Payments (Live)
- OpenAI (AI Agent)
- Resend (Email)

### الهوية
- الألوان: `#485869` (Slate) و `#34CC30` (Green)
- الخط: Tajawal
- الواجهة: عربية RTL

---

## ⛔ ممنوع لمس هذه الأنظمة (تعمل في الإنتاج)

1. **قاعدة البيانات** — لا تغير أي جدول موجود، لا تنشئ migrations جديدة بدون موافقة صريحة.
2. **جداول `users`, `orders`, `payment_sessions`, `reviews`, `transactions`** — لا تمسها.
3. **Serial IDs** — لا تحولها إلى UUID أبداً.
4. **Tap Payments Webhooks** — HMAC-SHA256 verification يعمل، لا تعدّله.
5. **منطق `search_freelancers`** — مرتّب بـ `rating DESC, avg_response_time ASC`، لا تغيره.
6. **بوت الواتساب ومنطق الـ intake** — كامل ويعمل.
7. **نظام حماية التسريب (Leak Guard)** — يعمل، لا تمسه.

## 🎯 المهمة المحددة

إعادة هيكلة **لوحة التحكم الإدارية فقط** (الواجهة + طبقة العرض). المطلوب:

### المشاكل الواجب حلها

1. **نصوص تقنية مكشوفة** مثل "Lifecycle"، "تشغيل إجباري"، "محدود لمرة بالساعة" → احذفها واستبدلها بلغة بشرية.

2. **تكرار الوظائف** في أقسام متعددة:
   - "النزاعات" موجودة كقسم مستقل + داخل مركز الدعم → ادمجهم.
   - "المحادثات" و "مركز الدعم" و "تصعيدات واتساب" → كلهم قناة واحدة.
   - "محاولات التسريب" قسم مستقل → ادمجه مع الدعم.

3. **صفحات شكلية بدون محرر**:
   - صفحة السياسات نص ثابت مع زر "رجوع" فقط → تحتاج محرر كامل.
   - Landing pages غير قابلة للتعديل → تحتاج Page Builder.

4. **نظام المشرفين لا يعمل**:
   - لا يمكن إضافة مشرف جديد.
   - لا يمكن تعديل صلاحياته.
   - لا يمكن تعديل كلمة المرور.
   - لا يوجد RBAC فعلي في الـ backend.

## 🏗️ الهيكل الجديد المطلوب — 9 أقسام فقط

```
📊 /admin                    → الرئيسية
💬 /admin/inbox              → صندوق الوارد الموحد (المحادثات + النزاعات + الدعم + التسريب)
👥 /admin/users              → المستخدمون (العملاء + المستقلون + المشرفون + الأدوار)
🛒 /admin/orders             → الطلبات والخدمات
💰 /admin/finance            → المالية
🎨 /admin/content            → المحتوى والصفحات (Page Builder)
📢 /admin/marketing          → التسويق والإشعارات
🤖 /admin/automation         → الأتمتة والذكاء الاصطناعي
⚙️ /admin/settings           → الإعدادات
```

**كل الأقسام الأخرى الموجودة حالياً** (30+ قسم) تُدمج كتبويبات فرعية داخل هذه التسعة.

## 📋 خطة التنفيذ — نفذها مرحلة بمرحلة

### المرحلة 1: إعادة هيكلة القائمة فقط (ابدأ بها)

**المطلوب:**
1. استبدل القائمة الجانبية بالأقسام التسعة.
2. انقل كل المحتوى الموجود إلى القسم الصحيح كتبويب فرعي.
3. احذف كل نص تقني مكشوف.
4. احتفظ بكل الصفحات الحالية (لا تحذفها)، فقط أعد ترتيبها.

**نتيجة هذه المرحلة:**
- قائمة جانبية نظيفة بـ 9 أقسام.
- كل صفحة موجودة سابقاً ما زالت تعمل لكن تحت قسمها الصحيح.
- لا نصوص تقنية مكشوفة.

**⚠️ لا تبدأ المرحلة 2 حتى أؤكد لك أن المرحلة 1 مكتملة.**

---

### المرحلة 2: دمج صندوق الوارد الموحد

**المطلوب:**
- صفحة `/admin/inbox` فيها 5 تبويبات: المحادثات، النزاعات، الدعم، تصعيدات واتساب، محاولات التسريب.
- تخطيط ثلاثي: قائمة يمين + محادثة وسط + تفاصيل يسار.
- مميزات: Assign to admin، تغيير الحالة، ملاحظات داخلية، ردود جاهزة، فلاتر.

---

### المرحلة 3: نظام المشرفين والصلاحيات (RBAC)

**المطلوب بناؤه من الصفر:**

1. **جداول قاعدة البيانات الجديدة** (بعد موافقتي):
   - `roles` (id, name, description, is_system)
   - `permissions` (id, key, label, group)
   - `role_permissions` (role_id, permission_id)
   - `admin_roles` (admin_id, role_id)

2. **الأدوار الافتراضية:**
   - Owner (المالك — صلاحيات مطلقة)
   - Admin (مدير عام)
   - Finance (مدير مالي)
   - Support (مشرف دعم)
   - Moderator (مراجع محتوى)
   - Accountant (محاسب للعرض فقط)

3. **صفحات المطلوب بناؤها:**
   - `/admin/users/admins` — قائمة المشرفين + إضافة جديد.
   - `/admin/users/admins/new` — شاشة إضافة مشرف.
   - `/admin/users/admins/[id]` — شاشة تعديل مشرف.
   - `/admin/users/roles` — إدارة الأدوار.
   - `/admin/users/roles/[id]` — تعديل صلاحيات دور.

4. **Middleware للـ backend:**
   - كل API endpoint يفحص صلاحية المستخدم قبل التنفيذ.
   - الاعتماد على إخفاء الواجهة فقط = ثغرة أمنية.

---

### المرحلة 4: محرر الصفحات (Page Builder)

**الأداة الموصى بها:** Puck (github.com/measuredco/puck) — مصمم خصيصاً لـ Next.js App Router.

**المطلوب:**
1. دمج Puck في `/admin/content`.
2. تحويل السياسات، الشروط، من نحن، FAQ إلى صفحات قابلة للتحرير.
3. بناء 3 قوالب Landing Pages جاهزة.
4. حفظ في جدول جديد `pages` (id, slug, title, content_json, seo_meta, status, created_at, updated_at, published_at).
5. جدول `page_revisions` للـ version history.
6. SEO fields لكل صفحة (title, description, og_image, slug, canonical).

## ⛔ قواعد صارمة أثناء العمل

### ممنوع منعاً باتاً:

1. **نصوص تقنية** بالإنجليزية في الواجهة: Lifecycle, Trigger, Webhook, Cron, Hash, Cache, Token, Endpoint, Payload, Queue — استبدلها بلغة عربية مبسطة.
   - استثناءات مسموحة: Meta, WhatsApp, Tap, Apple Pay, STC Pay, مدى.

2. **أزرار بدون وظيفة** — كل زر = API endpoint حقيقي.

3. **Placeholders** — الميزة كاملة أو لا تبدأها.

4. **تكرار وظيفة** في أكثر من قسم.

5. **مكتبتين أيقونات** — استخدم Lucide React فقط.

6. **كتابة بالإنجليزية** بنية الترجمة لاحقاً — اكتب بالعربية من البداية.

### مطلوب إلزامياً:

1. كل نص بالعربية الفصحى المبسطة.
2. كل صفحة Responsive (Mobile/Tablet/Desktop).
3. كل إجراء حساس يحتاج Confirmation Dialog.
4. استخدام الألوان الرسمية فقط (#485869 و #34CC30).
5. خط Tajawal للنصوص العربية.

## 🧪 خطة الاختبار — طبقها بعد كل مرحلة

قبل قول "الميزة مكتملة"، اختبر:

- [ ] ضغط كل الأزرار → كلها تعمل وتستدعي API حقيقياً.
- [ ] إنشاء سجل جديد → يُحفظ في قاعدة البيانات.
- [ ] تعديل سجل → التعديل يُحفظ ويظهر فوراً.
- [ ] حذف سجل → Soft delete مع إمكانية الاسترجاع.
- [ ] اختبار الصلاحيات → مشرف بدون صلاحية لا يرى القسم ولا API يقبل استدعاءه.
- [ ] Responsive → يعمل على 375px و 768px و 1280px.
- [ ] مراجعة النصوص → لا مصطلحات تقنية مكشوفة.
- [ ] السرعة → أول عرض < 1.5s.
- [ ] RTL → كل العناصر تعمل صحيح.

**إذا فشل اختبار واحد → الميزة غير مكتملة.**

## 🎯 الهدف النهائي

لوحة تحكم يستخدمها المالك (يوسف الغامدي) مباشرة بدون الحاجة لسؤال المطور عن أي شيء.

- كل زر له وظيفة.
- كل نص مفهوم.
- كل قسم في مكانه الصحيح.
- كل صلاحية تعمل فعلياً.
- لا تكرار. لا فوضى. لا نصوص تقنية.
