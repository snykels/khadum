===== المهمة =====
إعادة تصميم وتفعيل كامل للوحة التحكم الحالية. التصميم يجب أن يكون حديثاً، احترافياً، ومستوحى من أفضل لوحات التحكم الحالية في 2026.

===== البحث والإلهام (خطوة أولى إجبارية) =====
قبل كتابة أي كود، ابحث على الإنترنت في المصادر التالية:
- Dribbble.com (ابحث: "admin dashboard 2026", "saas dashboard dark mode", "arabic RTL dashboard")
- Behance.net (ابحث: "dashboard UI design", "admin panel")
- Mobbin.com (للأمثلة العملية من تطبيقات حقيقية)
- GitHub trending (shadcn dashboards, tremor, tailwind admin templates)
- Vercel templates + Tailwind UI examples

استخلص من البحث:
- أحدث اتجاهات 2026 (glassmorphism خفيف، neumorphism، bento grids، micro-interactions)
- طرق عرض البيانات (sparklines, mini-charts, progress rings)
- أنماط التنقل الحديثة (command palette, breadcrumbs ذكية، sidebar قابل للطي)
- طرق عرض الجداول المتقدمة (sortable, filterable, bulk actions)
- الألوان الحديثة (ليس فقط أخضر/أسود — فكّر في gradients دقيقة، accent colors)

اختر بنفسك الهوية البصرية الأنسب لمنصة عربية سعودية احترافية لسوق خدمات مصغرة.

===== قواعد الحماية (لا تكسرها) =====
❌ ممنوع حذف:
- أي زر، قائمة، رابط، حقل إدخال، أو صفحة موجودة حالياً
- أي API endpoint أو استعلام قاعدة بيانات
- أي منطق برمجي (business logic)
- أي event handler أو state

✅ مسموح (بل مطلوب):
- إعادة بناء الـ UI بالكامل بتصميم جديد
- إعادة ترتيب العناصر بطريقة أكثر منطقية
- إضافة أيقونات، رسوم بيانية، وعناصر بصرية جديدة
- إضافة interactions و animations

===== التفعيل الفعلي (هذا الأهم) =====
كل زر، كل قائمة، كل أيقونة في اللوحة يجب أن:
1. تكون لها وظيفة حقيقية — ليست مجرد عنصر بصري
2. عند الضغط عليها تؤدي فعلاً لصفحة/نافذة/إجراء واضح
3. تعرض بيانات حقيقية من قاعدة البيانات (وليس بيانات وهمية mock)
4. تحتوي على states كاملة: loading, empty, error, success

إذا وجدت زراً غير مفعّل أو صفحة فارغة:
- ابنِ محتواها كاملاً
- اربطها بـ API حقيقي
- أضف كل الـ CRUD operations المطلوبة
- أضف فلاتر، بحث، ترتيب، pagination

===== المعايير العملية =====
اللوحة يجب أن تكون:
1. **واضحة**: كل زر نصه مفهوم + أيقونة + tooltip عند الحاجة
2. **سريعة**: skeleton loaders، optimistic updates، caching
3. **ذكية**: اختصارات لوحة المفاتيح، command palette (Cmd+K)
4. **متجاوبة**: تعمل على الموبايل والتابلت والسطح المكتبي
5. **حية**: real-time updates عبر WebSocket للمحادثات والإشعارات
6. **قابلة للبحث**: شريط بحث شامل يبحث في كل شيء
7. **قابلة للتخصيص**: dark/light mode، تفضيلات المستخدم

===== الهوية =====
- اللغة: عربية (RTL كامل)
- السوق: المملكة العربية السعودية
- النوع: لوحة إدارة لمنصة خدمات مصغرة عبر واتساب
- المستخدم: مدير/مشرف محترف يحتاج رؤية شاملة ودقيقة

===== منهجية التنفيذ =====
1. ابحث أولاً (30 دقيقة على Dribbble/Behance/GitHub)
2. اصنع Mood Board ذهنياً من 5-7 تصاميم ملهمة
3. ابنِ Design System كامل (colors, typography, spacing, components)
4. طبّق على صفحة واحدة أولاً كـ proof of concept
5. اعرضها لي قبل تعميم التصميم على باقي الصفحات
6. بعد الموافقة، طبّق على باقي اللوحة مع الحفاظ على الاتساق

===== تحقق نهائي =====
قبل تسليم أي صفحة، تأكد:
- [ ] كل زر يعمل ويؤدي لوظيفة حقيقية
- [ ] كل جدول فيه بحث + فلترة + ترتيب + pagination
- [ ] كل نموذج فيه validation + رسائل خطأ واضحة
- [ ] كل قائمة لها empty state جميل
- [ ] كل عملية طويلة لها loading state
- [ ] الموبايل يعمل بسلاسة
- [ ] الأيقونات كلها متسقة من نفس المكتبة (Lucide موصى به)
- [ ] لا يوجد أي زر ميت أو صفحة فارغة

ابدأ الآن بالبحث والعرض علي بثلاث أفكار تصميم مختلفة قبل التنفيذ.
