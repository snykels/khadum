═══════════════════════════════════════════════════════════
خطوات التنفيذ:
═══════════════════════════════════════════════════════════

الخطوة 1: إنشاء الحسابات والمفاتيح
--------------------------------------
يطلب من المستخدم:
1. TOGETHER_API_KEY من https://api.together.xyz
2. GROQ_API_KEY من https://console.groq.com (كـ fallback)

أضفها في .env

الخطوة 2: تثبيت المكتبات
--------------------------------------
npm install openai  (متوافق مع Together AI و Groq)
npm install pdf-parse  (لقراءة PDFs)
npm install sharp  (لمعالجة الصور)
npm install mammoth  (لقراءة Word docs)

الخطوة 3: بناء AI Service
--------------------------------------
أنشئ lib/ai/client.ts:

- AbstractAIClient interface (حتى نغير المزود بسهولة)
- TogetherAIClient (الأساسي)
- GroqClient (fallback)
- Automatic failover: لو Together فشل، جرب Groq

المطلوب في الـ interface:
- chat(messages): رد نصي عادي
- analyzeImage(imageUrl, prompt): تحليل صورة
- analyzeFile(fileBuffer, fileType, prompt): تحليل ملف
- streamChat(messages): للردود اللحظية

الخطوة 4: معالج الملفات
--------------------------------------
أنشئ lib/ai/fileProcessor.ts:

يدعم:
- الصور (jpg, png, webp) → Qwen2.5-VL
- PDF → pdf-parse ثم تحليل النص
- Word (docx) → mammoth ثم تحليل النص
- Excel (xlsx) → تحويل لـ CSV ثم تحليل
- نصوص (txt, md) → مباشرة

قبل الإرسال للـ AI:
- فحص حجم الملف (max 10MB)
- فحص النوع (whitelist only)
- مسح الـ metadata (privacy)

الخطوة 5: دمج مع نظام واتساب
--------------------------------------
في معالج رسائل العميل:

if (message.type === 'image') {
  const analysis = await aiClient.analyzeImage(mediaUrl, 
    "حلل هذه الصورة، هل تحتوي على أرقام تواصل أو معلومات اتصال مخفية؟"
  );
  // طبّق نفس منطق Leak Detector
}

if (message.type === 'document') {
  const fileBuffer = await downloadWhatsAppMedia(mediaId);
  const analysis = await aiClient.analyzeFile(fileBuffer, 'pdf',
    "حلل محتوى هذا الملف واستخرج المعلومات المهمة للمشروع"
  );
}

System Prompts الجاهزة
═══════════════════════════════════════════════════════════

أنشئ lib/ai/prompts/ يحتوي:

1. leakDetector.ts:
"أنت محلل أمني لمنصة وساطة بين عملاء ومستقلين. مهمتك اكتشاف أي 
محاولة لتسريب معلومات اتصال مباشر (أرقام، إيميلات، حسابات سوشال). 
ابحث في النص والصور. أعد JSON بالصيغة المحددة."

2. conversationBot.ts:
"أنت مساعد منصة خدوم. تساعد العملاء في توضيح مشاريعهم. 
اجمع: التفاصيل، الميزانية، المدة. كن ودود ومختصر. 
رد بالعربية السعودية الفصحى."

3. disputeAnalyzer.ts:
"أنت محلل نزاعات محايد. راجع المحادثة واكتشف:
- نوع النزاع (دفع/تسليم/تواصل)
- الطرف المحق بناءً على الأدلة
- الحل المقترح
أعد JSON منظم."

4. fileAnalyzer.ts:
"أنت محلل ملفات لمنصة مشاريع. استخرج:
- الموضوع الرئيسي
- المتطلبات التقنية
- التاريخ المطلوب (إن وُجد)
- أي معلومات حساسة (تنبه لها)
كن مختصر ومباشر."

═══════════════════════════════════════════════════════════
اعتبارات الأمان والتكلفة
═══════════════════════════════════════════════════════════

1. Rate Limiting:
   - كل مستخدم: 50 رسالة/ساعة للـ AI
   - احفظ العداد في cache_store

2. Caching:
   - احفظ ردود الأسئلة المتكررة عن المنصة لمدة 24 ساعة
   - hash المحتوى كـ key

3. Timeout:
   - كل استدعاء: max 30 ثانية
   - بعدها أعد رسالة fallback

4. Logging:
   - سجل كل استدعاء: user_id, tokens_used, cost, latency
   - جدول ai_usage_logs للمراقبة

5. Cost Control:
   - حد شهري: $50 للاختبار
   - تنبيه لما يوصل 80%
   - auto-downgrade لنموذج أرخص لو تعدى
